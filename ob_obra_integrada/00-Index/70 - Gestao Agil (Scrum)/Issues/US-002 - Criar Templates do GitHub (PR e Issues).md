# US-002 - Criar Templates do GitHub (PR e Issues)

## User Story
As a **desenvolvedor do projeto**
I want to **ter templates padronizados para Pull Requests e Issues no GitHub**
So that **o fluxo de desenvolvimento, revisões de código e reporte de erros seja estruturado e organizado**

## Acceptance Criteria
- [ ] Criar o arquivo `.github/PULL_REQUEST_TEMPLATE.md` contendo seções para descrição das mudanças, tipo de alteração (feat, fix, etc.), checklist de testes e referências.
- [ ] Criar a pasta `.github/ISSUE_TEMPLATE/` com templates estruturados em YAML ou Markdown para relatos de bugs e sugestões de novas funcionalidades.

## Definition of Done
- [ ] Templates validados e criados na pasta `.github/`
- [ ] Commits integrados no repositório
