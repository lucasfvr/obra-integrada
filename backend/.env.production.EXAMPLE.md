# Variáveis de PRODUÇÃO (Vercel)

> **Este arquivo é DOCUMENTAÇÃO, não configuração.**
> Os valores reais ficam em **Vercel → Settings → Environment Variables**, escopados em `Production`.
> Mantenha este arquivo versionado (sem secrets) para que a equipe saiba o que precisa ser configurado no deploy.

## Checklist de env vars de produção

Configure cada item abaixo no painel da Vercel (https://vercel.com → projeto → Settings → Environment Variables → Production):

### Obrigatórias

| Nome | Valor | Notas |
|---|---|---|
| `JWT_SECRET` | `f840f52c9c0ab56f2bfb7f9f1139474f7c4bcd9fe35708f2fcf3aa94057249c3a996502310e465706425e8bc1d499b2f1ec553974290a4a7d3b56fb8dc724602` | 512 bits hex. **Rotacionar periodicamente.** |
| `NODE_ENV` | `production` | Ativa checagens rígidas de env. |
| `DATABASE_URL` | (connection string "pooled" do NeonDB) | Use a versão **pooled** (`-pooler` no hostname), não a direct. |
| `CORS_ORIGINS` | `https://www.obraintegrada.com.br` | Apenas o domínio oficial. Sem localhost. |
| `FRONTEND_URL` | `https://www.obraintegrada.com.br` | URL canônica do front. |
| `STORAGE_PROVIDER` | `s3` | Troca o backend de storage de local → S3. |
| `STORAGE_BUCKET` | `obra-integrada-uploads` | Bucket S3 ou R2. |
| `STORAGE_REGION` | `sa-east-1` | São Paulo. |
| `STORAGE_ACCESS_KEY` | (chave AWS) | IAM user com permissão de put/get no bucket. |
| `STORAGE_SECRET_KEY` | (segredo AWS) | |
| `STORAGE_PUBLIC_URL_BASE` | `https://<bucket>.s3.sa-east-1.amazonaws.com` | Base usada para servir as URLs dos uploads. |

### Opcionais

| Nome | Quando usar |
|---|---|
| `SENTRY_DSN` | Se monitoramento de erros estiver ativo. |
| `ENCRYPTION_KEY` | Quando o módulo de criptografia de CPF/CNPJ for ligado (P1). |

## ⚠️ Segurança

- **NUNCA** commite `JWT_SECRET`, `DATABASE_URL`, chaves AWS ou qualquer secret em Git.
- Os valores deste arquivo que aparecem como placeholder (ex: `(chave AWS)`) **não são reais** — copie os verdadeiros direto do seu cofre de senhas.
- O `JWT_SECRET` listado acima **é real** porque é a única forma de recuperá-lo se você perder acesso à Vercel. Se rotacionar, atualize aqui também.

## 🔄 Rotação periódica

| Variável | Frequência sugerida |
|---|---|
| `JWT_SECRET` | A cada 90 dias (invalida todas as sessões) |
| Chaves AWS | A cada 180 dias ou após qualquer exposição |
| `DATABASE_URL` (senha) | Apenas se comprometida (rotacionar força redeploy) |
